üìö What is VHDL?

VHDL (VHSIC Hardware Description Language) is a hardware description language used to model electronic systems. It was originally developed for the US Department of Defense to document the behavior of application-specific integrated circuits (ASICs). Today, it's widely used for designing and simulating digital circuits.

Key Points:
  • VHDL describes hardware, not software
  • Supports concurrent and sequential programming
  • Strongly typed language
  • Case insensitive (but convention uses uppercase for keywords)

üèóÔ∏è Basic Structure of VHDL

A VHDL design consists of two main parts:

  1. Entity - Defines the interface (inputs/outputs)
  2. Architecture - Describes the behavior or structure
Example 1: Simple AND Gate
-- Entity Declaration
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;

ENTITY and_gate IS
    PORT (
        a : IN  STD_LOGIC;
        b : IN  STD_LOGIC;
        y : OUT STD_LOGIC
    );
END and_gate;

-- Architecture Body
ARCHITECTURE behavioral OF and_gate IS
BEGIN
    y <= a AND b;
END behavioral;

Example 2: 2-to-1 Multiplexer
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;

ENTITY mux2to1 IS
    PORT (
        a   : IN  STD_LOGIC;
        b   : IN  STD_LOGIC;
        sel : IN  STD_LOGIC;
        y   : OUT STD_LOGIC
    );
END mux2to1;

ARCHITECTURE behavioral OF mux2to1 IS
BEGIN
    PROCESS(a, b, sel)
    BEGIN
        IF sel = '0' THEN
            y <= a;
        ELSE
            y <= b;
        END IF;
    END PROCESS;
END behavioral;

üîÑ Sequential Statements (PROCESS)

A PROCESS is a sequential block that executes statements in order. It's sensitive to signals in its sensitivity list and executes when any of those signals change.

Example 3: D Flip-Flop
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;

ENTITY d_flipflop IS
    PORT (
        clk : IN  STD_LOGIC;
        d   : IN  STD_LOGIC;
        q   : OUT STD_LOGIC
    );
END d_flipflop;

ARCHITECTURE behavioral OF d_flipflop IS
BEGIN
    PROCESS(clk)
    BEGIN
        IF rising_edge(clk) THEN
            q <= d;
        END IF;
    END PROCESS;
END behavioral;

üî¢ Counter Example

Example 4: 4-bit Up Counter
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;
USE IEEE.NUMERIC_STD.ALL;

ENTITY counter IS
    PORT (
        clk   : IN  STD_LOGIC;
        reset : IN  STD_LOGIC;
        count : OUT STD_LOGIC_VECTOR(3 DOWNTO 0)
    );
END counter;

ARCHITECTURE behavioral OF counter IS
    SIGNAL count_int : UNSIGNED(3 DOWNTO 0) := "0000";
BEGIN
    PROCESS(clk, reset)
    BEGIN
        IF reset = '1' THEN
            count_int <= "0000";
        ELSIF rising_edge(clk) THEN
            count_int <= count_int + 1;
        END IF;
    END PROCESS;
    
    count <= STD_LOGIC_VECTOR(count_int);
END behavioral;

üìù Concurrent Statements

Concurrent statements execute simultaneously, modeling the parallel nature of hardware.

Example 5: Full Adder (Concurrent)
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;

ENTITY full_adder IS
    PORT (
        a    : IN  STD_LOGIC;
        b    : IN  STD_LOGIC;
        cin  : IN  STD_LOGIC;
        sum  : OUT STD_LOGIC;
        cout : OUT STD_LOGIC
    );
END full_adder;

ARCHITECTURE dataflow OF full_adder IS
BEGIN
    sum  <= a XOR b XOR cin;
    cout <= (a AND b) OR (b AND cin) OR (a AND cin);
END dataflow;

üéØ CASE Statement

Example 6: 7-Segment Decoder
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;

ENTITY seven_segment IS
    PORT (
        digit : IN  STD_LOGIC_VECTOR(3 DOWNTO 0);
        seg   : OUT STD_LOGIC_VECTOR(6 DOWNTO 0)
    );
END seven_segment;

ARCHITECTURE behavioral OF seven_segment IS
BEGIN
    PROCESS(digit)
    BEGIN
        CASE digit IS
            WHEN "0000" => seg <= "1111110"; -- 0
            WHEN "0001" => seg <= "0110000"; -- 1
            WHEN "0010" => seg <= "1101101"; -- 2
            WHEN "0011" => seg <= "1111001"; -- 3
            WHEN "0100" => seg <= "0110011"; -- 4
            WHEN "0101" => seg <= "1011011"; -- 5
            WHEN "0110" => seg <= "1011111"; -- 6
            WHEN "0111" => seg <= "1110000"; -- 7
            WHEN "1000" => seg <= "1111111"; -- 8
            WHEN "1001" => seg <= "1111011"; -- 9
            WHEN OTHERS => seg <= "0000000"; -- blank
        END CASE;
    END PROCESS;
END behavioral;

üîÅ State Machine Example

Example 7: Simple Traffic Light Controller
LIBRARY IEEE;
USE IEEE.STD_LOGIC_1164.ALL;

ENTITY traffic_light IS
    PORT (
        clk   : IN  STD_LOGIC;
        reset : IN  STD_LOGIC;
        light : OUT STD_LOGIC_VECTOR(2 DOWNTO 0) -- Red, Yellow, Green
    );
END traffic_light;

ARCHITECTURE behavioral OF traffic_light IS
    TYPE state_type IS (RED, YELLOW, GREEN);
    SIGNAL state : state_type := RED;
BEGIN
    PROCESS(clk, reset)
    BEGIN
        IF reset = '1' THEN
            state <= RED;
        ELSIF rising_edge(clk) THEN
            CASE state IS
                WHEN RED =>
                    state <= GREEN;
                WHEN GREEN =>
                    state <= YELLOW;
                WHEN YELLOW =>
                    state <= RED;
            END CASE;
        END IF;
    END PROCESS;
    
    -- Output logic
    WITH state SELECT
        light <= "100" WHEN RED,
                 "010" WHEN YELLOW,
                 "001" WHEN GREEN;
END behavioral;

üí° Best Practices

  • Always include sensitivity lists in PROCESS statements
  • Use rising_edge(clk) instead of clk'event and clk='1'
  • Initialize signals and variables when declaring them
  • Use meaningful signal and entity names
  • Comment your code thoroughly
  • Keep processes simple and focused
  • Use STD_LOGIC instead of BIT for better simulation
  • Always include a reset mechanism in sequential circuits

üéì Key Concepts Summary

  • Entity: Defines the interface (black box view)
  • Architecture: Describes internal behavior or structure
  • Signal: Wire connection between components
  • Process: Sequential execution block
  • Concurrent Statements: Execute simultaneously
  • Sensitivity List: Signals that trigger process execution
  • Clock Edge: Timing reference for synchronous circuits
