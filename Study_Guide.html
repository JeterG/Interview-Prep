üìä Core Data Structures

Arrays

Key Operations:

Access: O(1) Search: O(n) Insert: O(n) Delete: O(n)

Common Patterns:

  • Two pointers (meeting in middle, sliding window)
  • Prefix sums for range queries
  • Kadane's algorithm for max subarray
‚úÖ Pros:
  • Cache-friendly
  • Constant access time
  • Memory efficient
‚ùå Cons:
  • Fixed size (usually)
  • Expensive insertions/deletions
  • Memory must be contiguous

Hash Tables

Key Operations:

Insert: O(1)* Delete: O(1)* Search: O(1)* *Average case, O(n) worst case

Common Applications:

  • Two Sum problems
  • Frequency counting
  • Caching/memoization
  • Detecting duplicates
‚úÖ Pros:
  • Excellent average performance
  • Flexible keys
  • Fast lookups
‚ùå Cons:
  • No ordering
  • Hash collisions possible
  • Space overhead

Trees (Binary Search Trees)

Key Operations:

Search: O(log n)* Insert: O(log n)* Delete: O(log n)* *For balanced trees, O(n) worst case if unbalanced

Common Traversals:

  • Inorder: Left ‚Üí Root ‚Üí Right (gives sorted order for BST)
  • Preorder: Root ‚Üí Left ‚Üí Right (tree copying)
  • Postorder: Left ‚Üí Right ‚Üí Root (tree deletion)
  • Level-order: BFS applications

Heaps (Priority Queues)

Key Operations:

Insert: O(log n) Extract-min/max: O(log n) Get-min/max: O(1) Heapify: O(n)

Common Applications:

  • K-th largest/smallest element
  • Merge K sorted lists
  • Dijkstra's algorithm
  • Median of stream

üéØ Essential Algorithm Patterns

Sliding Window

When to use: Contiguous subarray/substring problems

# Sliding Window Template
left = 0
for right in range(len(array)):
    # Expand window by including array[right]
    window_state += array[right]
    
    # Contract window while condition violated
    while invalid_condition:
        window_state -= array[left]
        left += 1
    
    # Update result with current window
    result = max(result, right - left + 1)

# Example: Longest substring with at most k distinct chars
def lengthOfLongestSubstringKDistinct(s, k):
    char_count = {}
    left = 0
    max_length = 0
    
    for right in range(len(s)):
        char_count[s[right]] = char_count.get(s[right], 0) + 1
        
        while len(char_count) > k:
            char_count[s[left]] -= 1
            if char_count[s[left]] == 0:
                del char_count[s[left]]
            left += 1
        
        max_length = max(max_length, right - left + 1)
    
    return max_length

Time: O(n) Space: O(1) or O(k)

Two Pointers

When to use: Sorted arrays, palindromes, pair problems

Variations:

  1. Opposite ends converging
  2. Same direction (fast/slow)
  3. Different arrays

Time: O(n) Space: O(1)

Binary Search

When to use: Sorted data, optimization problems

# Binary Search Template
left, right = 0, n - 1
while left <= right:
    mid = left + (right - left) // 2  # Prevents overflow
    
    if array[mid] == target:
        return mid
    elif array[mid] < target:
        left = mid + 1
    else:
        right = mid - 1

# Example: Find first bad version
def firstBadVersion(n):
    left, right = 1, n
    
    while left < right:
        mid = left + (right - left) // 2
        if isBadVersion(mid):
            right = mid  # First bad could be mid
        else:
            left = mid + 1
    
    return left

Time: O(log n) Space: O(1)

Dynamic Programming

When to use: Optimization problems with overlapping subproblems

Approaches:

  • Top-down (Memoization): Easier to write
  • Bottom-up (Tabulation): Better space optimization possible

Common Patterns:

  • 1D DP: Fibonacci, climbing stairs
  • 2D DP: Edit distance, longest common subsequence
  • State machine DP: Stock problems
# Example: Climbing Stairs (n steps, 1 or 2 at a time)
def climbStairs(n):
    if n <= 2:
        return n
    
    # Bottom-up DP
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2
    
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]

BFS / DFS (Graph Traversal)

When to use: Tree/graph problems, shortest path, level order

# DFS - Recursive (Stack via recursion)
def dfs(node, visited):
    if node in visited:
        return
    
    visited.add(node)
    print(node)  # Process node
    
    for neighbor in graph[node]:
        dfs(neighbor, visited)

# DFS - Iterative (Explicit stack)
def dfs_iterative(start):
    visited = set()
    stack = [start]
    
    while stack:
        node = stack.pop()
        
        if node not in visited:
            visited.add(node)
            print(node)
            
            # Add neighbors in reverse for same order as recursive
            for neighbor in reversed(graph[node]):
                if neighbor not in visited:
                    stack.append(neighbor)

# BFS - Uses queue for level-by-level traversal
from collections import deque

def bfs(start):
    visited = set([start])
    queue = deque([start])
    
    while queue:
        node = queue.popleft()
        print(node)
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

# BFS for shortest path in unweighted graph
def shortest_path_bfs(start, target):
    queue = deque([(start, [start])])
    visited = set([start])
    
    while queue:
        node, path = queue.popleft()
        
        if node == target:
            return path
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))
    
    return None

DFS Time: O(V+E) DFS Space: O(V)
BFS Time: O(V+E) BFS Space: O(V)

Two Pointers - Complete Examples

When to use: Sorted arrays, palindromes, pair problems

# Example 1: Two Sum (sorted array)
def twoSum(numbers, target):
    left, right = 0, len(numbers) - 1
    
    while left < right:
        current_sum = numbers[left] + numbers[right]
        
        if current_sum == target:
            return [left, right]
        elif current_sum < target:
            left += 1
        else:
            right -= 1
    
    return [-1, -1]

# Example 2: Valid Palindrome
def isPalindrome(s):
    left, right = 0, len(s) - 1
    
    while left < right:
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1
        
        if s[left].lower() != s[right].lower():
            return False
        
        left += 1
        right -= 1
    
    return True

# Example 3: Remove Duplicates (in-place)
def removeDuplicates(nums):
    if not nums:
        return 0
    
    left = 0
    for right in range(1, len(nums)):
        if nums[right] != nums[left]:
            left += 1
            nums[left] = nums[right]
    
    return left + 1

Time: O(n) Space: O(1)

Backtracking

When to use: Generate all solutions, permutations, combinations

# Backtracking Template
def backtrack(state, path, result):
    if is_solution(state):
        result.append(path[:])  # Make a copy
        return
    
    for choice in get_choices(state):
        # Make choice
        path.append(choice)
        make_choice(state, choice)
        
        # Recurse
        backtrack(state, path, result)
        
        # Undo choice (backtrack)
        path.pop()
        undo_choice(state, choice)

# Example: Generate all permutations
def permute(nums):
    result = []
    
    def backtrack(path, remaining):
        if not remaining:
            result.append(path[:])
            return
        
        for i in range(len(remaining)):
            backtrack(path + [remaining[i]], 
                     remaining[:i] + remaining[i+1:])
    
    backtrack([], nums)
    return result

Time: O(2^n) or O(n!) Space: O(n)

üìà Big-O Complexity Analysis

Time Complexities (Best to Worst)

Complexity Name Examples Performance
O(1) Constant Hash table lookup, array access Best
O(log n) Logarithmic Binary search, balanced BST ops Excellent
O(n) Linear Simple loop, linear search Good
O(n log n) Linearithmic Merge sort, heap sort Good
O(n¬≤) Quadratic Nested loops, bubble sort Fair
O(2^n) Exponential Recursive fibonacci, subsets Poor
O(n!) Factorial Permutation generation Worst

Space Complexity Considerations

  • Recursion stack space counts toward space complexity
  • In-place algorithms: O(1) extra space
  • Common trade-off: Use extra space for better time complexity
  • Consider auxiliary data structures (hash tables, arrays)

  • Meaningful variables
  • Handle edge cases

5. Test (5-7 mins)

  • Trace examples
  • Check edge cases
  • Discuss optimizations

‚öñÔ∏è Common Trade-offs to Discuss

Time vs Space

Example: Two Sum
  • Brute force: O(n¬≤) time, O(1) space
  • Hash table: O(n) time, O(n) space

Preprocessing vs Query Time

Example: Range Sum Queries
  • No preprocessing: O(n) per query
  • Prefix sums: O(n) preprocessing, O(1) per query

Accuracy vs Performance

Example: Finding Median
  • Exact: O(n log n) with sorting
  • Approximate: O(n) with quickselect
