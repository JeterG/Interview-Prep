1. C# & Language Fundamentals

Q: Explain the difference between value types and reference types
Value types store data directly in memory (int, bool, struct) while reference types store a reference to the memory location (classes, strings, arrays).
Q: What's the difference between an interface and an abstract class?
An interface defines a contract with no implementation (only method signatures), while an abstract class can have both implemented and abstract methods and can maintain state.
Q: How does garbage collection work in managed languages?
Garbage collection automatically reclaims memory by tracking object references and freeing objects that are no longer reachable from the program's root references.
Q: Explain async/await patterns
Async/await allows non-blocking asynchronous operations where async marks a function as asynchronous and await pauses execution until the awaited task completes without blocking the thread.
Q: Key differences between Python and C#?
C# is statically typed (type checking at compile-time), compiled to intermediate language, has explicit memory management through garbage collection, and requires semicolons, while Python is dynamically typed, interpreted, and uses indentation for structure.

2. GUI/Desktop Application Concepts

Q: How would you structure a desktop application that communicates with a backend API?
Use a layered architecture with separate layers for UI (presentation), business logic (services), and data access (API client), following the separation of concerns principle.
Q: Explain the MVC pattern
MVC separates application into Model (data/business logic), View (UI presentation), and Controller (handles user input and updates model/view).
Q: How do you handle state management in a GUI application?
Use a centralized state store or state management pattern (like Redux/Flux) where state changes flow unidirectionally and components subscribe to state updates.
Q: How do you prevent a GUI from freezing during long operations?
Run long operations on background threads or use async/await to keep the UI thread responsive, updating the UI with progress indicators.

Arrays & Strings

Reverse a string in-place
def reverse_string(s):
    # Convert to list (strings are immutable in Python)
    s = list(s)
    left, right = 0, len(s) - 1
    while left < right:
        s[left], s[right] = s[right], s[left]
        left += 1
        right -= 1
    return ''.join(s)

# Time: O(n), Space: O(n)
First non-repeating character
def first_unique_char(s):
    char_count = {}
    for char in s:
        char_count[char] = char_count.get(char, 0) + 1
    
    for i, char in enumerate(s):
        if char_count[char] == 1:
            return i
    return -1

# Time: O(n), Space: O(1) - max 26 letters

Hash Tables

Two Sum - Return indices that add to target
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

# Time: O(n), Space: O(n)
Group Anagrams
def group_anagrams(strs):
    from collections import defaultdict
    anagrams = defaultdict(list)
    
    for s in strs:
        # Sort string as key
        key = ''.join(sorted(s))
        anagrams[key].append(s)
    
    return list(anagrams.values())

# Time: O(n * k log k), Space: O(n*k)
# n = number of strings, k = max length of string

Linked Lists

Detect cycle in linked list
def has_cycle(head):
    if not head:
        return False
    
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False

# Floyd's Cycle Detection (Tortoise & Hare)
# Time: O(n), Space: O(1)
Reverse a linked list
def reverse_list(head):
    prev = None
    current = head
    
    while current:
        next_node = current.next
        current.next = prev
        prev = current
        current = next_node
    
    return prev

# Time: O(n), Space: O(1)

Trees

Binary tree level order traversal (BFS)
def level_order(root):
    if not root:
        return []
    
    from collections import deque
    result = []
    queue = deque([root])
    
    while queue:
        level_size = len(queue)
        level = []
        
        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)
            
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        
        result.append(level)
    
    return result

# Time: O(n), Space: O(n)
Validate Binary Search Tree
def is_valid_bst(root):
    def validate(node, min_val, max_val):
        if not node:
            return True
        
        if node.val <= min_val or node.val >= max_val:
            return False
        
        return (validate(node.left, min_val, node.val) and
                validate(node.right, node.val, max_val))
    
    return validate(root, float('-inf'), float('inf'))

# Time: O(n), Space: O(h) where h is height

Real-time Sensor Data Monitor
from collections import deque
from datetime import datetime, timedelta

class SensorMonitor:
    def __init__(self, threshold):
        self.threshold = threshold
        self.readings = deque()  # (value, timestamp)
    
    def add_reading(self, value, timestamp):
        self.readings.append((value, timestamp))
    
    def get_average(self, time_window):
        """Get average over last time_window seconds"""
        cutoff = datetime.now() - timedelta(seconds=time_window)
        recent = [val for val, ts in self.readings if ts >= cutoff]
        return sum(recent) / len(recent) if recent else 0
    
    def check_alert(self):
        """Check if last reading exceeds threshold"""
        if self.readings and self.readings[-1][0] > self.threshold:
            return True
        return False
Parse error logs from file
import re
from datetime import datetime

def parse_error_logs(filepath):
    """Extract errors with timestamps from log file"""
    errors = []
    pattern = r'\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\] ERROR: (.+)'
    
    with open(filepath, 'r') as f:
        for line in f:
            match = re.match(pattern, line)
            if match:
                timestamp = datetime.strptime(match.group(1), 
                                             '%Y-%m-%d %H:%M:%S')
                message = match.group(2).strip()
                errors.append({'timestamp': timestamp, 'message': message})
    
    return errors
Validate medical device configuration
def validate_config(config_dict):
    """Validate device configuration meets requirements"""
    required = ['device_id', 'voltage', 'frequency', 'enabled']
    
    # Check all required fields present
    if not all(key in config_dict for key in required):
        return False, "Missing required fields"
    
    # Validate device_id: alphanumeric, 8 chars
    device_id = config_dict['device_id']
    if not (len(device_id) == 8 and device_id.isalnum()):
        return False, "Invalid device_id"
    
    # Validate voltage: float, 0-10V
    try:
        voltage = float(config_dict['voltage'])
        if not (0 <= voltage <= 10):
            return False, "Voltage out of range"
    except (ValueError, TypeError):
        return False, "Invalid voltage"
    
    # Validate frequency: int, 1-100Hz
    try:
        freq = int(config_dict['frequency'])
        if not (1 <= freq <= 100):
            return False, "Frequency out of range"
    except (ValueError, TypeError):
        return False, "Invalid frequency"
    
    # Validate enabled: boolean
    if not isinstance(config_dict['enabled'], bool):
        return False, "Invalid enabled field"
    
    return True, "Valid configuration"

SELECT device_id, COUNT(*) as error_count
FROM error_logs
WHERE timestamp >= NOW() - INTERVAL 24 HOUR
GROUP BY device_id
ORDER BY error_count DESC
LIMIT 10;
Q: How would you optimize a slow query?
Add indexes on frequently queried columns, use EXPLAIN to analyze query execution, avoid SELECT *, limit result sets, and consider query restructuring or denormalization for complex joins.
Q: Explain indexing and when to use it
Indexes are data structures (usually B-trees) that speed up data retrieval by creating pointers to data locations; use them on columns frequently used in WHERE, JOIN, and ORDER BY clauses.
Q: SQL vs NoSQL - when to use each?
Use SQL for structured data with complex relationships and ACID transactions; use NoSQL for unstructured/semi-structured data, horizontal scaling needs, and flexible schemas.

6. Concurrency & Threading

Q: How do you prevent race conditions?
Use locks/mutexes to synchronize access to shared resources, ensure atomic operations, or use thread-safe data structures.
Q: Threading vs async/await?
Threading uses multiple OS threads for parallel execution (CPU-bound tasks), while async/await uses cooperative multitasking on a single thread for I/O-bound operations without blocking.

7. Error Handling & Testing

Q: How do you write testable code?
Use dependency injection, separate concerns into single-responsibility functions, avoid global state, and design functions with clear inputs/outputs.
Q: Unit vs Integration vs E2E tests?
Unit tests verify individual functions in isolation, integration tests verify multiple components working together, and E2E tests verify entire user workflows through the system.

def process_device_data(data):
    """Process device readings with error handling"""
    if not data:
        return []

8. System Design Concepts

Q: Design a medical device GUI showing real-time patient data
Use WebSocket connection for real-time updates, implement data validation layer, add connection retry logic with exponential backoff, cache last known good state, and display clear error states when connection fails.
Q: Polling vs WebSockets for real-time data?
WebSockets provide true real-time bidirectional communication with lower latency and overhead, while polling repeatedly requests updates at intervals (simpler but less efficient).
Q: How do you ensure software reliability in critical systems?
Implement comprehensive error handling, extensive automated testing, redundancy and failover mechanisms, detailed logging/monitoring, and graceful degradation when components fail.
